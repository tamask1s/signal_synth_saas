# Synsigra product UX audit — 2026-07-10

## Outcome

The product already exposes the complete account, pack, generation, job, verifier,
scenario, custom-pack and API-key workflows. The main usability gap is not missing
capability but continuity: the landing page, application shell and post-action
destinations currently feel like separate products.

## Findings

1. The public landing page uses a polished dark, glass-like visual system, while
   the application uses a generic light admin layout. This weakens trust at the
   moment a visitor enters the actual product.
2. The application has no persistent product header or obvious account/profile
   entry point. A flat side navigation mixes the guided workflow, expert tools,
   account settings and API documentation at the same level.
3. Successful actions often leave the user where the action started. In
   particular, job creation prints a raw response instead of opening and focusing
   the newly created job.
4. Authentication does not preserve a useful return destination. Email
   verification and password reset end on the account page instead of continuing
   the first-use workflow.
5. Client-side navigation does not consistently reset scroll position or carry a
   short success message to the destination page.
6. Empty, loading and error states are accurate but do not always offer the next
   useful action.
7. User-visible copy mixes `Synsigra`, `SynSigRa`, `Sinsigra` and path-like names.
   Only `Synsigra` should be used as the product name. Technical identifiers such
   as `/syn_sig_ra` and Apache directive names remain unchanged for compatibility.

## Information architecture

The persistent header exposes the five common destinations: Workspace, Packs,
Generate, Jobs and Verify. Account is always available as a profile action. The
secondary navigation groups destinations as:

- Workflow: Start, Packs, Generate, Jobs, Verify
- Build custom tests: Scenarios, Custom packs
- Settings: Account, Developer/API

This keeps the normal algorithm-QA path prominent without hiding advanced tools.

## Transactional navigation rules

- Registration: remain on Account with an explicit “check your inbox” state.
- Email verification: sign in and continue to Packs for the first challenge.
- Sign-in: continue to the validated `next` destination, or Workspace.
- Password reset: sign in and continue to Workspace.
- Pack selection/custom-pack creation: continue to Generate with the selection.
- Job creation: continue to Jobs, focus the new job and show a success message.
- Completed job: continue to Verify with that job selected.
- Sign-out: continue to Account and confirm the result.
